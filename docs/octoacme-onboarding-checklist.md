# OctoAcme — New Team Member Onboarding Checklist

Use this checklist when a new team member joins a project to ensure they're set up for success quickly and consistently. See [Roles and Personas](octoacme-roles-and-personas.md) for role definitions.

---

## Before Day 1 (PM / Lead responsible)

- [ ] Access provisioned: GitHub org, project board, communication channels (Slack/Teams)
- [ ] Add to recurring meetings (standup, sprint planning, demo, retro)
- [ ] Assign an onboarding buddy from the team
- [ ] Share this checklist with the new team member

---

## Week 1 — Orientation

### Project Context
- [ ] Read [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read [Roles and Personas](octoacme-roles-and-personas.md) and identify your role
- [ ] Review the Project One-pager (linked in project repo/README)
- [ ] Review the current Roadmap and Release Plan

### Process Docs
- [ ] Review [Project Planning](octoacme-project-planning.md) — understand backlog and DoD
- [ ] Review [Execution & Tracking](octoacme-execution-and-tracking.md) — understand team rhythm and PR workflow
- [ ] Review [Risk Management & Communication](octoacme-risks-and-communication.md) — understand how to raise blockers

### Tools & Workflow
- [ ] Clone the project repo and run it locally (or review setup docs in repo README)
- [ ] Review open issues and current sprint board
- [ ] Submit your first PR (even a docs fix) to practice the PR workflow

---

## Week 2 — Active Participation

- [ ] Attend and contribute to at least one standup, sprint planning, and demo
- [ ] Take ownership of at least one backlog item
- [ ] Review the [Risk Review Checklist](octoacme-risk-review-checklist.md) and understand how to raise risks
- [ ] Review the [Release & Deployment Guide](octoacme-release-and-deployment.md)

---

## Handoff Points to Know

| Situation | Who to Contact | How |
|---|---|---|
| Blocked on a task | Raise in daily standup; escalate to PM if >1 day | Standup / Slack |
| Unclear requirements | Contact PM or Product Manager | Direct message / backlog comment |
| Found a risk or dependency | Add to risk register; flag in next standup | Risk register / Standup |
| Need access or tooling help | Ask onboarding buddy or PM | Slack |
| Ready for code review | Open PR, link issue, request review from buddy | GitHub PR |

---

## 30-Day Check-In (PM + new team member)

- [ ] Comfortable with the team rhythm and ceremonies?
- [ ] Clear on your role and responsibilities?
- [ ] Any gaps in access, tooling, or documentation?
- [ ] Feedback captured for improving this onboarding checklist?
